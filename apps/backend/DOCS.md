# MealAI — Backend Docs (Supabase)

---

## Overview

The backend is fully powered by **Supabase** — no custom server required.

| Feature | How it's handled |
|---|---|
| Authentication | Supabase Auth (email/password, OAuth) |
| Database | Supabase PostgreSQL |
| REST API | Auto-generated by Supabase (PostgREST) |
| Custom logic | Supabase Edge Functions (Deno) |
| Security | Row Level Security (RLS) on every table |
| Session storage | `expo-secure-store` on device |

---

## Day 1 — Supabase Project Setup

**Install Supabase CLI:**
```bash
npm install -g supabase
```

**Initialize Supabase in the backend folder:**
```bash
cd apps/backend
supabase init
```

**Login and link to your project:**
```bash
supabase login
supabase link --project-ref your-project-id
```

**Start local Supabase dev environment:**
```bash
supabase start
# Gives you a local URL, anon key, and Studio at http://localhost:54323
```

**Stop local environment:**
```bash
supabase stop
```

### Tables

**`profiles`** — extends `auth.users`
```sql
id             UUID  (FK → auth.users)
name           TEXT
avatar_url     TEXT
goal           TEXT  ('lose_weight' | 'gain_muscle' | 'maintain')
diet_type      TEXT  ('None' | 'Vegetarian' | 'Vegan' | 'Keto' | 'Paleo' | 'Gluten-Free')
allergies      TEXT[]
calorie_target INT
created_at     TIMESTAMPTZ
updated_at     TIMESTAMPTZ
```
→ Auto-created via trigger when a new user signs up (`on_auth_user_created`)

---

**`meals`** — meal catalog
```sql
id           UUID  (PK)
title        TEXT
image        TEXT
calories     INT
protein      INT
carbs        INT
fat          INT
prep_time    INT
category     TEXT  ('Breakfast' | 'Lunch' | 'Dinner' | 'Snack' | 'Healthy')
tags         TEXT[]
description  TEXT
ingredients  TEXT[]
instructions TEXT[]
created_at   TIMESTAMPTZ
```
→ Pre-seeded with 8 meals in the migration file

---

**`saved_meals`** — user bookmarks
```sql
id        UUID  (PK)
user_id   UUID  (FK → profiles)
meal_id   UUID  (FK → meals)
saved_at  TIMESTAMPTZ
UNIQUE(user_id, meal_id)
```

---

**`daily_logs`** — calorie tracking
```sql
id         UUID  (PK)
user_id    UUID  (FK → profiles)
meal_id    UUID  (FK → meals)
meal_time  TEXT  ('Breakfast' | 'Lunch' | 'Dinner' | 'Snack')
logged_at  DATE
calories   INT
protein    INT
created_at TIMESTAMPTZ
```

---

**`chat_messages`** — chat history
```sql
id         UUID  (PK)
user_id    UUID  (FK → profiles)
text       TEXT
sender     TEXT  ('user' | 'ai')
created_at TIMESTAMPTZ
```

## Day 1 — Row Level Security (RLS)

Every table has RLS enabled. Users can only access their own data.

| Table | Policy |
|---|---|
| `profiles` | SELECT / UPDATE own row only (`auth.uid() = id`) |
| `meals` | SELECT allowed for all authenticated users |
| `saved_meals` | SELECT / INSERT / DELETE own rows only |
| `daily_logs` | SELECT / INSERT / DELETE own rows only |
| `chat_messages` | SELECT / INSERT own rows only |

## Day 1 — Edge Function (Chat)

**Location:** `supabase/functions/chat/index.ts`
**Runtime:** Deno (not Node.js)

**What it does:**
1. Receives `{ message, userId }` in the request body
2. Generates an AI response (mock for now — swap with OpenAI/Claude later)
3. Saves both user message and AI reply to `chat_messages` table
4. Returns `{ reply }` to the client

**Deploy to Supabase:**
```bash
supabase functions deploy chat
```

**Test locally:**
```bash
supabase functions serve chat
curl -X POST http://localhost:54321/functions/v1/chat \
  -H "Content-Type: application/json" \
  -d '{"message": "What should I eat for dinner?", "userId": "your-user-id"}'
```

## Day 1 — Auto-generated REST API

Supabase auto-generates a REST API (PostgREST) for all tables. No routes needed.

**Examples using the Supabase JS client:**

```ts
import { supabase } from '@/lib/supabase';

// Get all meals
const { data } = await supabase.from('meals').select('*');

// Search meals
const { data } = await supabase
  .from('meals')
  .select('*')
  .ilike('title', '%chicken%');

// Get saved meals for current user
const { data } = await supabase
  .from('saved_meals')
  .select('*, meals(*)')
  .eq('user_id', userId);

// Save a meal
await supabase.from('saved_meals').insert({ user_id: userId, meal_id: mealId });

// Unsave a meal
await supabase.from('saved_meals').delete().match({ user_id: userId, meal_id: mealId });

// Log a meal
await supabase.from('daily_logs').insert({
  user_id: userId,
  meal_id: mealId,
  meal_time: 'Lunch',
  calories: 320,
  protein: 42,
});

// Get today's logs
const { data } = await supabase
  .from('daily_logs')
  .select('*, meals(title)')
  .eq('user_id', userId)
  .eq('logged_at', new Date().toISOString().split('T')[0]);
```

## Day 1 — Auth

Supabase Auth is used directly — no custom auth endpoints needed.

```ts
import { supabase } from '@/lib/supabase';

// Sign up
await supabase.auth.signUp({ email, password, options: { data: { name } } });

// Sign in
await supabase.auth.signInWithPassword({ email, password });

// Sign out
await supabase.auth.signOut();

// Get current user
const { data: { user } } = await supabase.auth.getUser();

// Listen to auth state changes
supabase.auth.onAuthStateChange((event, session) => {
  console.log(event, session);
});
```

On signup, the `on_auth_user_created` trigger automatically creates a `profiles` row.

## Environment Variables

Copy `.env.example` to `.env`:
```
SUPABASE_URL=https://your-project-id.supabase.co
SUPABASE_ANON_KEY=your-anon-key
SUPABASE_SERVICE_ROLE_KEY=your-service-role-key
```

> `SERVICE_ROLE_KEY` bypasses RLS — only use it in Edge Functions or server-side code, never in the mobile app.
